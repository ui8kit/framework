## О проекте UI8Kit Framework

Это **комплексная UI система**, которая предоставляет как React компоненты, так и семантические HTML5/CSS3 утилиты. Проект построен как **монопепозиторий** с использованием Turbo для оркестрации.

### Архитектура проекта

**Приложения:**
- `@ui8kit/vite-local` - локальная среда разработки для компонентов, лейаутов и документации
- `@ui8kit/generator` - оркестратор генерации статических сайтов

**Пакеты:**
- `@ui8kit/render` - рендерер React компонентов в статический HTML с `data-class` атрибутами
- `@ui8kit/generator` - координирует полный пайплайн генерации (вью → CSS → HTML → ассеты)

### Ключевые возможности

1. **React компоненты** с типизированными utility props
2. **HTML5/CSS3 семантика** в стиле Bootstrap/Uikit3 (например, `button button-primary`)
3. **Utility Props System** со строгой валидацией через `utility-props.map.ts`
4. **Component Variants** только для button, badge, card, grid, image

### Workflow разработки

1. **Локальная разработка** - `@ui8kit/vite-local` для разработки компонентов
2. **Статическая генерация** - `bun run generate` генерирует чистый HTML из React роутов
3. **Веб-публикация** - деплой HTML файлов из `dist/html/`

### Пайплайн генерации

1. **Генерация вью** - рендеринг React компонентов в HTML с `data-class` → `.liquid` файлы
2. **Генерация CSS** - парсинг вью → извлечение классов → генерация `@apply` CSS + чистый CSS3
3. **Сборка HTML** - комбинация `.liquid` вью с лейаутами → финальные HTML страницы
4. **Копирование ассетов** - статические файлы в выходные директории

### Важные правила (из системного промпта)

**Декларативные правила UI8Kit:**

1. **Только одиночные значения в пропах** (маппинг на Tailwind): `col="span-1"` → `col-span-1`
2. **Запрет className** на всех компонентах кроме Grid (только для responsive модификаторов)
3. **Utility props как источник истины** - все визуальные свойства из `utility-props.map.ts`
4. **Обязательные `data-class` атрибуты** для семантических селекторов CSS

**Ограничения рендеринга:**
- Компоненты рендерятся напрямую без контекст-провайдеров
- Нет RouterProvider, ThemeProvider во время генерации
- Роутер конфиг должен использовать `createBrowserRouter` с `children` массивом

**Недавний рефакторинг:**
- Убрана секция `render` из конфига
- Рендеринг стал проще - прямой рендеринг компонентов
- Все пути берутся из конфигурации
