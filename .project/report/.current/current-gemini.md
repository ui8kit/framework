# Текущий контекст сессии и состояние проекта

## 1. Системный контекст (System Prompt)
Я работаю как AI-ассистент в среде Cursor с моделью `gemini-3-pro-preview`.
Мои инструкции включают:
- **Инструменты**: Полный доступ к файловой системе, терминалу, поиску (grep/glob), Git, и MCP серверам (браузер, контекст пользователя).
- **Стиль общения**: Лаконичный, без эмодзи (если не просят), использование Markdown для кода.
- **Правила кодинга**:
  - Всегда читать файлы перед редактированием.
  - Использовать `read_lints` после правок.
  - Следовать существующему стилю кода.
  - Не удалять изменения пользователя без спроса.
- **Управление задачами**: Использование `todo_write` для сложных задач.

## 2. Обзор проекта (ui8kit-framework)
Это монорепозиторий (Monorepo) на базе **Turborepo**, реализующий дизайн-систему и фреймворк **ui8kit**.

### Структура
- **`packages/ui8kit`**: Ядро дизайн-системы.
  - Компоненты (`Block`, `Stack`, `Group`, `Button` и др.).
  - Варианты стилей (CVA).
  - Утилиты для маппинга пропсов в Tailwind классы.
- **`apps/web`**: Основное веб-приложение, использующее библиотеку.
  - Содержит "Блоки" (`blocks/`) — крупные секции страниц (Hero, Features).
  - Элементы (`elements/`) и страницы (`routes/`).
- **`apps/engine`**: Движок (вероятно, для генерации или управления контентом).
- **`packages/generator`**: Инструменты для генерации CSS/HTML.

### Архитектурные правила (из .cursor/rules)
- **Semantic Props**: Вместо `className` используются пропсы (напр., `gap="6"`, `p="4"`, `bg="primary"`).
- **Компоненты компоновки**: Использование `Stack` (flex col), `Group` (flex row), `Grid` вместо `div`.
- **Data-attributes**: Обязательное использование `data-class` для генерации CSS.
- **Запрет**: Inline styles и прямые классы Tailwind (где возможно).
